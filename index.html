Do prompt ao commit:
criando jogos no navegador com Claude Code

Como um setup mínimo no browser virou uma fábrica pessoal de protótipos jogáveis — sem IDE, sem build step, sem fricção.

  • 🌐 Ambiente 100% no navegador — Claude Code via claude.ai/code, conectado direto ao meu repositório no GitHub. Sem instalar IDE, SDK ou toolchain.
  • 🎮 Stack dos jogos — HTML + JavaScript puro + Canvas/WebGL (e WebXR para VR). Roda em qualquer browser; basta abrir o index.html.
  • ⚡ Fluxo — descrevo a ideia em linguagem natural → Claude gera e edita os arquivos → testo no preview → peço ajustes → commit e push pro GitHub pela própria interface.
  • 🔁 Iteração em minutos — ciclo curtíssimo entre "quero X" e ver funcionando. Bugs resolvidos na conversa, sem sair do contexto.
  • 🧠 Papel que assumi — mais game designer / product owner do que programador. Eu defino mecânica, balanceamento e feel; o Claude implementa e refatora.

Resultados

4 protótipos jogáveis,
um fim de semana de experimentação

Cada jogo nasceu de uma conversa. Todos rodam no navegador, todos versionados no GitHub.

  • 🏰 Dark Tower · Tower Defense — grid com waves crescentes, múltiplas torres com upgrades, economia de ouro e pathfinding dos inimigos. Foco em balanceamento e progressão.
  • 💀 Doomzinho · FPS 3D — homenagem ao clássico Doom: renderização 3D no Canvas, mouse-look, inimigos com IA simples e sistema de tiro/vida.
  • 🥽 Doomzinho VR · WebXR — versão imersiva rodando em headset pelo browser, via WebXR API. Mesma base de código, adaptada para controles de movimento e câmera estereoscópica.
  • 🚀 Star Fox-like · Rail Shooter 3D — voo "on-rails" estilo SNES, mira livre, ondas de inimigos e obstáculos — com aquele charme low-poly retrô.

Aprendizado principal: com Claude Code + GitHub no navegador, a barreira entre "ter uma ideia de jogo" e "ter o jogo rodando e versionado" praticamente desaparece. O gargalo passa a ser criatividade, não execução.
